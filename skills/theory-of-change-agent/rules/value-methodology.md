## 0. The concept chain + the four principles

**Chain:** `사회문제 → 사회적 가치(=성과/Outcome) → 지표 → 산출식 → 보고서`.
- **사회문제 → 사회적 가치:** 구체적으로 정의된 **사회문제가 활동을 통해 해결된 크기** = 사회적 가치.
- **사회적 가치 → 지표:** 그 사회적 가치를 관리·제고하려고 **측정 지표**를 세운다.
- **지표 → 산출식:** 한 지표에 여러 **산출식**이 가능. 조직이 실제 확보 가능한 데이터 = **가용 데이터**.
  적용성이 높은 것 = **표준식**, 나머지 = **응용식(변형)**. 표준식 × 가용 데이터 → **측정 결과** → 보고서.

**Four principles (mutually complementary; when they conflict, the EARLIER one wins):**
1. **[실행력] Actionability** — provide information actually **useful to manage and raise** the social value.
2. **[설명력] Explanatory power** — provide information that **explains** the real change.
3. **[일관성] Consistency** — provide it consistently (comparable over time/across cases).
4. **[보수성] Conservatism** — provide **conservative, not over-stated** estimates.
   *(Reference: GRI reporting principles; and SROI "do not over-claim".)*

## I. 사회문제 (Social problem)

**Basic definition.** Not every 문제 is a 사회문제. A 사회문제 is a state that is **simultaneously**:
- **① 구조적 (structural):** the cause arises **social-structurally** → **cannot self-correct by individual
  effort**. (If one person's effort could fix it, it is an individual problem, not a 사회문제.)
- **② 현저하게 다수 (widespread):** it applies to / is recognized by a **notable majority**, not a small
  minority.
- **③ 고통받는 상태 (harm):** absent intervention, it produces **serious deficiency(결핍) and violation(침해)**.

**Operational definition — considerations:**
- Nail the **현상 → 원인 (phenomenon → cause)** causal link; the solution must target the **cause**.
- Do a light **사회문제 맵핑** (who suffers, why structurally, what deficiency).

---

## II. 사회적 가치 (Social value = the Outcome/성과)

**Basic definition.** Not every 가치 is 사회적 가치. **사회적 가치 = the SIZE by which the 사회문제 is
resolved**, and it is **distinct from general 효용가치**.
- The total positive utility an org creates is **not** all social value.
- **Neutral-0 lens:** put the neutral state at **0**. Only the improvement **from the harmed state (−) up
  to neutral (0)** counts as 사회적 가치.
- Improvement **above 0** (however positive/useful) is **NOT** 사회적 가치 (it's general utility / 공공·효용
  가치). Use this to distinguish 사회적 가치 from mere utility expansion.
- *(This is the counterfactual/additionality principle in practice: only the change that recovers the
  problem counts; explain it by tying the outcome to the problem's cause.)*

**Operational definition — considerations:**
- Not every created social value is **measurable/manageable**.
- Social value is inherently **complex and qualitative**; measuring it wholesale is impossible.
  - Analogy: a fruit's "맛있는 정도" depends on 당도·식감·향·풍미 — you can't measure all of it.
- So pick **one representative characteristic** and measure that (measure 당도 → express as **Brix** to
  stand in for "맛"). Likewise pick a **핵심(core) 사회적 가치** to measure, distinct from **보조(secondary)**.
- This is exactly the **Outcome(성과)** in the results chain: a **change-of-state**, not output volume
  (`koica-rules.md` §3.4, rule A08).

## III. 지표 (Social-value measurement indicator)

**Basic definition.** An indicator is a **척도(measure)** for the social value the org wants to manage.
Basic shape: **"'솔루션'을 통한 '어떤 현상'의 '어떤 변화분'"** (the change, brought about by the solution, in
some phenomenon).

**Definition — considerations.** Define an indicator that fits the operationally-defined social value; via
its value / derivation it should satisfy:
- **(1) 실행력의 원칙:** can you judge whether the org is doing well and **what to improve**?
- **(2) 설명력의 원칙:** does it have **explanatory power** for the **real** change the org made?

**Indicator type axes (choose deliberately):**
- 개입 **사전(A) vs 사후(B)** 비교 vs **변화분(A−B)** 자체.
- **단위당** vs **총량**.
- **화폐가치** vs **비화폐가치**.
- **정량** vs **정성**.

(Maps onto `koica-rules.md` A01 SMART / A02 CREAM.)

## IV–V. 산출식 & 보고 (brief)
- **산출식:** 표준식 vs 응용식(변형); **가용 데이터**로 실제 측정. (measurement-plan detail, not a gate.)
- **측정 결과·보고:** 대상 기간 / 창출 성과(결과값) / 보고 시점(공시 시점); 보고서(개요·산출방식·분석·제언).
  Aligns with IMP reporting.
