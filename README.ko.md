# 변화이론 에이전트 (Theory of Change Agent)

[English](README.md) · **한국어** · [日本語](README.ja.md) · [Tiếng Việt](README.vi.md)

임팩트 실무자라면 누구나 아는 순간이 있습니다. 화면에는 빈 성과표, 눈앞에는 제출 마감 —
그런데 어느 칸부터 채워야 할지 막막한 순간. 논리는 머릿속에 있는데, 양식이 그걸 받아주질 않습니다.

**변화이론 에이전트는 그 논리를 꺼내 주는 AI 인터뷰어입니다.** 한 번에 하나씩, 편한 말로 질문에
답하면 결과사슬 — 사회문제 → 활동 → 산출물 → 성과 → 영향 — 을 함께 세우고, 국제개발 사업이면
**KOICA 가이드라인에 맞는 PDM(성과체계표)** 을, 임팩트 스타트업·사회공헌·비영리 사업이면
**변화이론 도식**을 만들어 줍니다. 클로드 같은 AI 비서 위에서 동작하고, 한국어로 물으면 한국어로
답합니다.

칸만 채워 주는 게 아닙니다. "그건 성과가 아니라 산출물의 확대입니다"라고 **바로잡아 주고**,
자기 작업을 **증명**합니다: 결정적 품질 게이트는 자체 벤치마크에서 **18/18 검출**, 모든 규칙에
**KOICA 가이드라인(36쪽) 쪽수 인용**이 붙어 있으며, 예산 모듈은 실제 KOICA형 예산서 2종을
**원단위까지 재현**했습니다.

**이런 분들을 위한 도구입니다:**

- KOICA류 사업에 지원하며 **PDM을 처음 쓰는 분** — 빈 양식 대신 구조화된 질문
- **초안이 이미 있는 분** — 심사위원보다 먼저 가이드라인 대조 점검
- 아이디어만 있는 **창업가·CSR·비영리 팀** — 임팩트 논리를 한 장으로

## 빠른 시작

1. 설치 (30초 — 아래 참고)
2. 이렇게 말하기: *"변화이론 에이전트로 PDM 만들어줘"* — 또는 사업계획서/연차보고서 첨부
3. 질문에 답하다가, 마음에 들면 **"확정"**
4. `out/pdm.md`를 열어 보세요. 계속 쓸지 여기서 판단하시면 됩니다.

## 실제로 이렇게 동작합니다

```
나:      변화이론 에이전트로 PDM 만들어줘.
에이전트: 어떤 일을 하고 계신가요?
         ① 국제개발협력(PDM)  ② 임팩트 스타트업
         ③ 사회공헌(CSR, ESG) ④ 비영리
나:      ① — 라오스 모자보건 사업이야.
에이전트: [질문 1 · 사회문제] 이 사업이 해결하려는 문제는 무엇인가요 —
         누가 고통받고 있고, 원인은 무엇인가요?
나:      농촌 산모들이 집에서 분만해. 보건소가 멀고, 못 믿기도 하고.
에이전트: 사회문제 요건에 부합합니다 (구조적 · 다수 · 실제 고통).
         현상: 높은 가정분만율. 원인: 시설 접근성과 신뢰 부족.
         그렇다면 겨냥할 성과는 상태의 변화 — "시설분만 실천 정착"
         입니다. "교육 횟수 확대"가 아니라요. 이 틀로 갈까요?
         …
         [한 번에 한 질문: 활동 → 산출물 → 지표 → 가정 —
          답할 때마다 결과사슬에 연결]
         …
나:      확정
에이전트: ✔ 품질 게이트 통과 (핵심 규칙 8/8).
         ⚠ 성과 2는 산출물의 확대로 읽힙니다 — "교육 확대"는 횟수를
           세는 것이지 행동 변화가 아닙니다. 수정 제안: …
         ◇ 지표 1-1의 IRIS+ 근접 지표 제안 (참고용 · 공식 매핑 아님)
         최종 결과: out/pdm.md
```

에이전트는 *고통*을 듣고, 원인을 짚고, 산출물이 성과 행세를 하는 걸 거부했습니다.
심사에서 늘 지적받는 바로 그 교정이 — 제출 전에 일어납니다.

## 어떻게 동작하나요

**인터뷰 → 결과사슬 구축 → 검증 → 게이트 → 렌더.** 각 단계가 다음 단계로 이어집니다.

인터뷰는 변화이론을 따릅니다: 제대로 정의된 **사회문제**(구조적 · 다수 · 실제 고통)에 닻을 내리고,
현상 → 원인을 추적하며, 모든 성과가 **원인을 회복하는 상태의 변화**일 것을 요구합니다 — 산출물의
확대도, 일반 효용도 아니고요. 조직 단위 문서(비영리 연차보고서, 스타트업 사업계획서)는 프로젝트
감지 단계를 거칩니다: **전체 프로젝트**(미션↔프로젝트 연결 구조도)로 볼지, **특정 프로젝트**
하나로 들어갈지 선택합니다.

| 산출물 | 내용 |
|---|---|
| `pdm.md` | KOICA 4×4 PDM 매트릭스 — 영향/성과/산출물/활동 × 요약/지표/검증수단/중요가정 |
| `toc.md` | 변화이론 노드 도식 (Mermaid 미지원 뷰어용 텍스트 인과 흐름 병기) + 임팩트 측정을 위해 지금부터 모아야 할 데이터 |
| `monitoring.md` | 지표별 측정 계획: 정의·산출식·기초치·목표치·출처·시기·수집주체·분리 기준 |
| `budget.md` *(선택)* | PDM 연동 예산서: 활동별 세목, 산출근거(단가×수량×횟수), 분담, 연차별 합계 |
| `details/pdm.json` | 단일 원본 — 위의 모든 뷰가 여기서 렌더링됩니다 |

입력: 일반 대화, PDF, 그리고 **한글 파일(.hwp/.hwpx)** — 내장 추출기는 의존성이 없어 앱
샌드박스에서도 동작합니다.

## 직접 확인할 수 있는 검증

"AI를 믿으세요"는 없습니다. 검사는 코드이고, 증거는 레포 안에 있습니다:

- **결정적 품질 게이트** — KOICA 핵심 규칙 8개(영향 지표 금지, 산출물 3~4개, 지표 수 상한,
  검증수단 필수, 고아 노드 금지, 산출물 명사형…)를 순수 파이썬으로 강제.
  [`benchmark/`](./skills/theory-of-change-agent/benchmark/)의 위반 삽입 벤치마크에서
  **검출 정확도 18/18**.
- **쪽수 인용 규칙** — [`koica-rules.md`](./skills/theory-of-change-agent/rules/koica-rules.md)의
  모든 규칙이 KOICA 공식 가이드라인 쪽수를 인용합니다. 저희가 아니라 원문과 논쟁하세요.
- **성과 2중 검증** — 논리 검사(그 변화가 원인을 회복하는가?) + 글로벌 임팩트 지표 라이브러리
  **IRIS+ 593개 지표** 근접 매칭 (참고용, © GIIN).
- **예산 계산은 스크립트가, AI는 절대 안 함** — 모든 합계·비율·분담·일반관리비 상한을 결정적으로
  계산·검증합니다. 실제 KOICA형 예산서 2종으로 3회 검증 — 총계를 **원단위까지 재현**했고, 그
  과정에서 실제 규칙 버그 3건을 잡아 고쳤습니다.
- 권장 규칙(SMART, CREAM, 성별 분리지표)은 점수로만 알려 드립니다 — 취향의 결정권은 당신에게.

## 설치 — 30초

**준비물:** Claude (Claude Code · 데스크톱 앱 · claude.ai 중 무엇이든) · `python3`
(데스크톱/웹 코드 실행 샌드박스에 기본 포함)

### Claude Code (자동 업데이트, 권장)

```
/plugin marketplace add IMPACT-SQUARE/theory-of-change-agent
/plugin install theory-of-change-agent@impact-square
```

업데이트는 `/plugin update theory-of-change-agent`.

### Claude 데스크톱 · Antigravity · claude.ai (zip 업로드)

1. 스킬을 zip으로 압축 (또는 만들어져 있는 `theory-of-change-agent.zip` 사용):
   ```bash
   cd skills && zip -r theory-of-change-agent.zip theory-of-change-agent \
     -x '*/.DS_Store' -x '*/out/*' -x '*/.omc/*' -x '*/__pycache__/*'
   ```
2. 업로드: **Claude 데스크톱** `Settings → Skills → Add → Upload` · **Antigravity** Skills 화면 ·
   **claude.ai** `Settings → Capabilities → Skills → Upload`. 유료 플랜 + 코드 실행 켜기 필요.
3. 채팅에 *"변화이론 에이전트로 KOICA PDM 만들어줘"* 라고 쓰면 시작됩니다.

> 앱에 올린 스킬은 자동 업데이트되지 않습니다 — 바뀌면 새 zip을 다시 올리세요. Antigravity에서
> 변화이론 도식이 코드로 보일 때의 해결법(Open VSX에서 `bierner.markdown-mermaid` 설치, 또는
> 병기된 텍스트 인과 흐름 읽기) 포함 상세 안내:
> [`INSTALL-desktop.md`](./skills/theory-of-change-agent/INSTALL-desktop.md).
