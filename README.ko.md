# 변화이론 에이전트 (Theory of Change Agent)

[English](README.md) · **한국어** · [日本語](README.ja.md) · [Tiếng Việt](README.vi.md)

성과표는 사업을 구상하는 일보다 작성하는 일이 더 어려울 때가 많습니다. 문제가 무엇인지, 무엇을
할지, 어떤 변화를 만들려는지는 알고 있어도, 이를 논리가 맞는 PDM으로 옮기는 것은 별개의 일입니다.

**변화이론 에이전트는 이 작업을 대화로 안내합니다.** 질문을 하나씩 던지고, 답변을 모아
결과사슬(사회문제 → 활동 → 산출물 → 성과 → 영향)을 만듭니다. 국제개발협력 사업이라면 **KOICA
가이드라인에 맞는 PDM(성과체계표)** 을, 임팩트 스타트업·사회공헌·비영리 사업이라면 **변화이론
도식**을 만들어 줍니다. 클로드 같은 AI 비서 위에서 동작하며, 사용자가 쓰는 언어로 답합니다.

단순히 양식을 채우는 도구가 아닙니다. 산출물을 성과처럼 적어 둔 것 같은 논리의 빈틈을 짚어 주고,
그 점검 과정은 추적할 수 있습니다. 결정적 품질 게이트는 검출 벤치마크에서 **18/18**을 기록했고,
각 규칙은 **KOICA 가이드라인(36쪽)의 해당 쪽수**를 인용하며, 예산 모듈은 KOICA형 실제 예산서
2종의 총계를 **원 단위까지 재현**했습니다.

**이런 분들에게 유용합니다.**

- KOICA 등 공모 사업에 지원하며 **PDM을 처음 작성하는 분**
- 초안을 들고 **심사 전에 가이드라인과 대조해 보고 싶은 분**
- 초기 임팩트 모델을 정리해야 하는 **창업가, CSR·비영리 팀**

## 빠른 시작

1. 스킬을 설치합니다. 30초 정도 걸립니다.
2. *"변화이론 에이전트로 PDM 만들어줘"* 라고 말합니다. 사업계획서나 연차보고서를 첨부해도 됩니다.
3. 질문에 답하고, 내용이 정리되면 **"확정"** 이라고 말합니다.
4. `out/`에서 결과를 확인합니다. 국제개발협력은 `pdm.md`, 그 외에는 `toc.md`입니다.

## 내 상황에 맞는 경로

인터뷰는 문제에서 출발해 원인을 찾고, 목표로 하는 변화를 측정 가능한 형태로 다듬습니다.
최종 결과물은 용도에 따라 달라집니다.

| 상황 | 만드는 결과물 | 시작하기 좋은 자료 |
|---|---|---|
| 국제개발협력 사업 | KOICA 가이드라인 PDM | 아이디어, 제안서, 기존 PDM 초안 |
| 임팩트 스타트업 또는 신규사업 | 변화이론 도식 | 사업계획서 또는 해결하려는 문제 |
| CSR 또는 ESG 사업 | 변화이론 도식 | 사업 개요서 또는 제안서 |
| 비영리 프로그램 또는 연차보고서 | 변화이론 도식 | 연차보고서. 조직 전체 또는 특정 프로젝트 선택 |

새 PDM은 확정 전에 품질 게이트를 통과해야 합니다. 이미 승인된 PDM은 감사 모드로 점검할 수
있으며, 문서를 고치지 않고 가이드라인과 다른 지점만 보고합니다.

## 실제 동작 예시

```
나:      변화이론 에이전트로 PDM 만들어줘.
에이전트: 어떤 일을 하고 계신가요?
         ① 국제개발협력(PDM)  ② 임팩트 스타트업
         ③ 사회공헌(CSR, ESG) ④ 비영리
나:      ①. 라오스 모자보건 사업이야.
에이전트: [질문 1 · 사회문제] 이 사업이 해결하려는 문제는 무엇인가요?
         누가 어려움을 겪고 있고, 원인은 무엇인가요?
나:      농촌 산모들이 집에서 분만해. 보건소가 멀고, 못 믿기도 하고.
에이전트: 사회문제로 볼 수 있습니다. 구조적이고, 다수에게 영향을 미치며,
         실제 피해가 있습니다. 드러난 현상은 높은 가정분만율이고,
         원인은 시설 접근성과 신뢰 부족입니다. 그렇다면 성과는
         "더 많은 산모가 시설에서 분만한다"처럼 상태의 변화로 잡는 것이
         좋습니다. "교육 횟수 확대"가 아니라요. 의도하신 변화와 맞나요?
         …
         [활동, 산출물, 지표, 가정을 하나씩 묻고,
          답할 때마다 결과사슬에 연결합니다.]
         …
나:      확정
에이전트: ✔ 품질 게이트 통과 (핵심 규칙 8/8).
         ⚠ 성과 2는 산출물의 확대에 가깝습니다. "교육 확대"는 횟수를
           세는 것이지 행동 변화가 아닙니다. 수정 제안: …
         ◇ 지표 1-1에 가까운 IRIS+ 지표를 제안합니다.
           참고용이며 공식 매핑은 아닙니다.
         결과: out/pdm.md
```

인터뷰는 활동·산출물과 사업이 만들려는 변화를 구분합니다. 심사에서 흔히 지적되는 문제를
제출 전에 드러내는 데 도움이 됩니다.

## 어떻게 동작하나요

**인터뷰 → 결과사슬 구축 → 검증 → 게이트 → 렌더.** 각 단계는 앞 단계의 작업을 이어받습니다.

인터뷰는 변화이론의 방식을 따릅니다. 구조적이고, 다수에게 영향을 미치고, 실제 피해를 낳는
**사회문제**를 먼저 정의하고, 드러난 현상과 원인을 연결한 뒤, 그 원인을 해소하는 변화를 성과로
정의합니다. 산출물이나 막연한 편익 서술은 성과로 다루지 않습니다.

연차보고서나 사업계획서처럼 조직 단위 문서를 넣으면, 문서가 하나의 프로젝트를 다루는지 여러 개를
다루는지 먼저 확인합니다. **조직 전체**를 미션과 프로젝트의 연결 구조로 볼 수도 있고, **특정
프로젝트** 하나로 들어갈 수도 있습니다.

| 산출물 | 내용 |
|---|---|
| `pdm.md` | KOICA 4×4 PDM 매트릭스: 영향/성과/산출물/활동 × 요약/지표/검증수단/중요가정 |
| `toc.md` | 변화이론 도식. Mermaid 미지원 뷰어를 위한 텍스트 버전과, 임팩트 측정을 위해 모아야 할 데이터 안내 포함 |
| `details/monitoring.md` | 지표별 측정 계획: 정의, 산출식, 기초치, 목표치, 출처, 시기, 수집 주체, 분리 기준 |
| `budget.md` *(선택)* | PDM 연동 예산서: 활동별 세목, 산출근거(단가×수량×횟수), 분담, 연차별 합계 |
| `details/pdm.json` | 위의 모든 뷰를 렌더링하는 원본 데이터 |

대화만으로 시작해도 되고, PDF나 **한글 파일(.hwp/.hwpx)** 을 넣어도 됩니다. 내장 추출기는 외부
의존성이 없어 앱 샌드박스에서도 동작합니다.

## 직접 확인할 수 있는 검증

점검은 코드로 구현되어 있고, 이 저장소에서 확인할 수 있습니다.

- **결정적 품질 게이트:** 영향 지표 금지, 산출물 3~4개, 지표 수 상한, 검증수단 필수, 고아 노드
  금지, 산출물 명사형 등 KOICA 핵심 규칙 8개를 순수 파이썬으로 검사합니다.
  [위반 삽입 벤치마크](./skills/theory-of-change-agent/benchmark/)에서 **검출 정확도 18/18**을
  기록했습니다.
- **쪽수 인용 규칙:** [`koica-rules.md`](./skills/theory-of-change-agent/rules/koica-rules.md)의
  모든 규칙이 KOICA 공식 가이드라인의 해당 쪽수를 인용합니다.
- **성과 검토:** 의도한 변화가 원인을 실제로 해소하는지 논리적으로 확인합니다. 성과 지표는
  **IRIS+ 임팩트 지표 593개** 중 가장 가까운 지표와 대조합니다. 참고용이며 공식 매핑은 아닙니다.
- **예산 계산은 스크립트로:** 모든 합계, 비율, 분담, 일반관리비 상한을 스크립트가 계산하고
  검증합니다. KOICA형 실제 예산서 2종으로 3회 검증해 총계를 원 단위까지 재현했고, 이 과정에서
  규칙 결함 3건을 찾아 고쳤습니다.
- **권장 규칙:** SMART, CREAM, 성별 분리지표는 점수로만 보고합니다. 반영 여부는 사용자가 정합니다.

## 설치: 30초

**준비물:** Claude Code, Claude 데스크톱, claude.ai 중 하나와 `python3`. 데스크톱과 웹의 코드 실행
샌드박스에는 파이썬이 기본으로 들어 있습니다.

### Claude Code (자동 업데이트, 권장)

```
/plugin marketplace add IMPACT-SQUARE/theory-of-change-agent
/plugin install theory-of-change-agent@impact-square
```

업데이트는 `/plugin update theory-of-change-agent`.

### Claude 데스크톱 · Antigravity · claude.ai (zip 업로드)

1. 스킬을 zip으로 압축합니다. 미리 만들어진 `theory-of-change-agent.zip`이 있으면 그대로 씁니다.
   ```bash
   cd skills && zip -r theory-of-change-agent.zip theory-of-change-agent \
     -x '*/.DS_Store' -x '*/out/*' -x '*/.omc/*' -x '*/__pycache__/*'
   ```
2. 업로드합니다. **Claude 데스크톱**은 `Settings → Skills → Add → Upload`, **Antigravity**는 Skills
   화면, **claude.ai**는 `Settings → Capabilities → Skills → Upload`. 유료 플랜과 코드 실행 기능이
   필요합니다.
3. 채팅에 *"변화이론 에이전트로 KOICA PDM 만들어줘"* 라고 쓰면 시작됩니다.

> 앱에 올린 스킬은 자동으로 업데이트되지 않습니다. 내용이 바뀌면 새 zip을 다시 올리세요.
> Antigravity에서 도식이 코드로 보이면 Open VSX에서 `bierner.markdown-mermaid`를 설치하거나,
> 함께 출력되는 텍스트 버전을 읽으면 됩니다. 상세 안내:
> [`INSTALL-desktop.md`](./skills/theory-of-change-agent/INSTALL-desktop.md).

## 데이터 정책

- `docs/`에는 공개 문서만 있습니다 (KOICA PDM 가이드라인, Theory of Change 자료).
- `benchmark/`의 PDM은 가상 데이터입니다. 구조는 유지하되 실명과 실제 금액은 제거했습니다.
