# Theory of Change Agent

[English](README.md) · [한국어](README.ko.md) · **日本語** · [Tiếng Việt](README.vi.md)

インパクト実務者なら誰もが知っている瞬間があります。画面には空白の成果マトリックス、目の前には
提出締切 — それなのに、どのセルから埋めればいいのか分からない瞬間。ロジックは頭の中にあるのに、
様式がそれを受け取ってくれません。

**Theory of Change Agent は、そのロジックを引き出す AI インタビュアーです。** 一問ずつ、普通の
言葉で答えていくと、結果連鎖 — 社会問題 → 活動 → アウトプット → アウトカム → インパクト — を
一緒に組み立て、国際開発事業なら **KOICA ガイドライン準拠の PDM（Project Design Matrix）** を、
インパクトスタートアップ・CSR・非営利なら**セオリー・オブ・チェンジ図**を作成します。Claude などの
AI アシスタント上で動作し、話しかけた言語で答えます。

セルを埋めるだけではありません。「それはアウトカムではなく、アウトプットの拡大です」と**指摘し**、
自分の仕事を**証明**します: 決定論的な品質ゲートは自社ベンチマークで **18/18 検出**、すべての
ルールに **KOICA ガイドライン（36 ページ）のページ引用**が付き、予算モジュールは実際の KOICA 型
予算書 2 種を**ウォン単位まで再現**しました。

**こんな方のためのツールです:**

- KOICA 型事業に応募して **PDM を初めて書く方** — 空白の様式の代わりに構造化された質問を
- **ドラフトが既にある方** — 審査員より先にガイドライン照合チェックを
- アイデアだけの**起業家・CSR・非営利チーム** — インパクトのロジックを 1 枚に

## クイックスタート

1. インストール（30 秒 — 下記参照）
2. こう話しかける: *「Theory of Change Agent で PDM を作って」* — または事業計画書/年次報告書を添付
3. 質問に答え、納得できたら**「確定」**
4. `out/pdm.md` を開いてみてください。続けるかどうかは、そこで判断できます。

## 実際の動き

```
あなた:  Theory of Change Agent で PDM を作って。
Agent:  どんなお仕事ですか？
        ① 国際開発協力（PDM） ② インパクトスタートアップ
        ③ CSR / ESG          ④ 非営利
あなた:  ① — ラオスの母子保健事業です。
Agent:  [質問 1 · 社会問題] この事業が解決しようとする問題は何ですか —
        誰が苦しんでいて、原因は何でしょう？
あなた:  農村部の妊産婦が自宅分娩しています。保健施設が遠く、
        信頼もされていません。
Agent:  社会問題の要件を満たします（構造的 · 多数 · 実際の苦痛）。
        現象: 高い自宅分娩率。原因: 施設へのアクセスと信頼の不足。
        では目指すアウトカムは状態の変化 — 「施設分娩の実践が定着する」
        です。「研修回数の拡大」ではなく。この枠組みで進めますか？
        …
        [一問ずつ: 活動 → アウトプット → 指標 → 前提 —
         答えるたびに結果連鎖へ接続]
        …
あなた:  確定
Agent:  ✔ 品質ゲート通過（重要ルール 8/8）。
        ⚠ アウトカム 2 はアウトプットの拡大に読めます — 「教育の拡大」は
          回数を数えるもので、行動変化ではありません。修正案: …
        ◇ 指標 1-1 の IRIS+ 近接指標を提案（参考 · 公式マッピングではない）
        最終結果: out/pdm.md
```

エージェントは*苦痛*を聞き、原因を特定し、アウトプットがアウトカムのふりをするのを拒みました。
審査で必ず指摘されるあの修正が — 提出前に起こります。

## 仕組み

**インタビュー → 結果連鎖の構築 → 検証 → ゲート → レンダリング。** 各ステップが次へつながります。

インタビューはセオリー・オブ・チェンジに従います: きちんと定義された**社会問題**（構造的 · 多数 ·
実際の苦痛）を錨とし、現象 → 原因をたどり、すべてのアウトカムが**原因を回復する状態の変化**である
ことを要求します — アウトプットの拡大でも、一般的な効用でもなく。組織単位の文書（非営利の年次
報告書、スタートアップの事業計画書）はプロジェクト検出ステップを通ります: **全体プロジェクト**
（ミッション↔プロジェクトの構造図）で見るか、**特定のプロジェクト**に絞るかを選べます。

| 成果物 | 内容 |
|---|---|
| `pdm.md` | KOICA 4×4 PDM マトリックス — インパクト/アウトカム/アウトプット/活動 × 要約/指標/検証手段/重要な前提 |
| `toc.md` | セオリー・オブ・チェンジのノード図（Mermaid 非対応ビューア用のテキスト因果フロー併記）+ インパクト測定のために今から集めるべきデータ |
| `monitoring.md` | 指標ごとの測定計画: 定義・算出式・ベースライン・目標値・出所・時期・収集主体・分解基準 |
| `budget.md` *(任意)* | PDM 連動の予算書: 活動別細目、算出根拠（単価×数量×回数）、分担、年次別合計 |
| `details/pdm.json` | 単一のソース — 上のすべてのビューはここからレンダリングされます |

入力: 通常の会話、PDF、そして**韓国語 HWP ファイル（.hwp/.hwpx）** — 同梱の抽出器は依存関係ゼロ
なので、アプリのサンドボックスでも動きます。

## 自分で確かめられる検証

「AI を信じてください」はありません。チェックはコードで、証拠はリポジトリの中にあります:

- **決定論的品質ゲート** — KOICA 重要ルール 8 個（インパクトに指標を置かない、アウトプット 3〜4 個、
  指標数の上限、検証手段必須、孤児ノード禁止、アウトプットは名詞形…）を純 Python で強制。
  [`benchmark/`](./skills/theory-of-change-agent/benchmark/) の違反埋め込みベンチマークで
  **検出精度 18/18**。
- **ページ引用ルール** — [`koica-rules.md`](./skills/theory-of-change-agent/rules/koica-rules.md)
  のすべてのルールが KOICA 公式ガイドラインのページを引用。私たちではなく、原文と議論してください。
- **アウトカム二重検証** — 論理チェック（その変化は原因を回復するか？）+ グローバルなインパクト
  指標ライブラリ **IRIS+ の 593 指標**との近接マッチング（参考、© GIIN）。
- **予算計算はスクリプトが行い、AI は決して行わない** — すべての合計・比率・分担・一般管理費上限を
  決定論的に計算・検証。実際の KOICA 型予算書 2 種で 3 回検証 — 合計を**ウォン単位まで再現**し、
  その過程で実際のルールバグを 3 件発見・修正しました。
- 推奨ルール（SMART、CREAM、性別分解指標）はスコアで報告するだけ — センスの決定権はあなたに。

## インストール — 30 秒

**必要なもの:** Claude（Claude Code · デスクトップアプリ · claude.ai のいずれか）· `python3`
（デスクトップ/Web のコード実行サンドボックスに標準搭載）

### Claude Code（自動更新、推奨）

```
/plugin marketplace add IMPACT-SQUARE/theory-of-change-agent
/plugin install theory-of-change-agent@impact-square
```

更新は `/plugin update theory-of-change-agent`。

### Claude デスクトップ · Antigravity · claude.ai（zip アップロード）

1. スキルを zip 化（またはビルド済みの `theory-of-change-agent.zip` を使用）:
   ```bash
   cd skills && zip -r theory-of-change-agent.zip theory-of-change-agent \
     -x '*/.DS_Store' -x '*/out/*' -x '*/.omc/*' -x '*/__pycache__/*'
   ```
2. アップロード: **Claude デスクトップ** `Settings → Skills → Add → Upload` · **Antigravity**
   Skills 画面 · **claude.ai** `Settings → Capabilities → Skills → Upload`。有料プラン + コード
   実行オンが必要。
3. チャットに *「Theory of Change Agent で KOICA PDM を作って」* と書けば始まります。

> アプリにアップロードしたスキルは自動更新されません — 変更時は新しい zip を再アップロードして
> ください。Antigravity で図がコードのまま表示されるときの解決法（Open VSX から
> `bierner.markdown-mermaid` をインストール、または併記のテキスト因果フローを読む）を含む詳細:
> [`INSTALL-desktop.md`](./skills/theory-of-change-agent/INSTALL-desktop.md)。

## データポリシー

- `docs/` には公開文書のみ（KOICA PDM ガイドライン、Theory of Change 資料）。
- `benchmark/` の PDM は架空 — 実名・実額は除去し、構造のみ保持。
- 実際の事業 PDM・予算書の原本はこのリポジトリには決して置きません。
