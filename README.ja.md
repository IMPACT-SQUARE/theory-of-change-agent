# セオリー・オブ・チェンジ エージェント (Theory of Change Agent)

[한국어](README.md) · [English](README.en.md) · **日本語** · [Tiếng Việt](README.vi.md)

事業の成果構造を**セオリー・オブ・チェンジ（変化の理論）**で整理するツールです。会話のように質問に
答えていくと、国際開発協力事業なら **PDM（Project Design Matrix、プロジェクト・デザイン・マトリックス）**
を、インパクトスタートアップ・企業の社会貢献・非営利事業なら**セオリー・オブ・チェンジ図**を作成します。
Claude などの AI アシスタント上で動作します。

空白の表を前に「何から書けばいいのか」と途方に暮れる時間を減らすことが目的です。一問ずつの質問に
答えていくだけで、KOICA（韓国国際協力団）PDM ガイドラインに沿った表が完成します。日本語で答えれば
結果も日本語で出力されます（入力言語に合わせて出力されます）。

---

## 何を手伝ってくれるのか

- **セオリー・オブ・チェンジ**の流れ（投入 → 活動 → アウトプット → アウトカム → インパクト）に沿って
  質問し、答えを整理しながら結果連鎖（リザルトチェーン）を一緒に組み立てます。
- 完成すると **PDM 表**と**モニタリング表**の 2 つの文書を作成します。
- 作成された**アウトカムが本物のアウトカムかを二重に検証**します — ① 対象の変化が問題の原因を
  回復しているかを問う**論理検証**、② グローバルなインパクト指標ライブラリ **IRIS+ の 593 指標**から
  最も近い指標を参考として提示する**近接指標マッチング**。
- 最後に KOICA ガイドラインの主要ルールを守っているか**自動チェック**するため、審査でよく指摘される
  ミスを事前に防げます。

こんな方に役立ちます。

- KOICA 事業に応募するために初めて PDM を書く方
- 既にドラフトがあり、ガイドラインに合っているか確認したい方
- アイデアだけがあり、成果構造がまだ固まっていない方

## 始め方

このツールは開始時に 3 つのことを順番に尋ねます。

**(1) 何をしようとしていますか？** 4 つから選びます — **国際開発協力（PDM）**なら PDM 表を、
**インパクトスタートアップ（新規事業開発）**・**社会貢献（CSR, ESG）**・**非営利**なら
セオリー・オブ・チェンジ図を作成します。背後のロジックは同じで、最終成果物の形だけが異なります。
（5 つ目の「インパクト投資家（投資審査）」は今後開発予定です。）

非営利やインパクトスタートアップのように**組織単位の文書**（事業計画書、年次報告書）を入れて複数の
プロジェクトが検出された場合は、**全体プロジェクト**（ミッションとプロジェクトのつながりの構造図）で
見るか、**特定のプロジェクト** 1 つに絞るかを尋ねます。

**(2) 今どんな状態ですか？**

- **アイデアだけあります** — 構想はあるが、まとまった資料はまだ。
- **事業計画書・ドラフト・既存の PDM などの文書があります** — そのファイルを読んで活用します。
  （承認済みの PDM なら、変更せずチェックだけ受けることもできます。）

**(3) どう進めますか？**

- **質問に一つずつ答えながらじっくり** — 最も丁寧です。（アイデアから約 10–20 分・文書から約 5–10 分）
- **成果物を先に作ってから直す** — 最速です（約 2–5 分）。まずドラフトを作成し、気になる箇所を直して
  「確定」と言えば品質チェックを経て完成します。

---

## 何が作られるのか

作業が終わると、成果物がファイルとして保存されます。

- **PDM 表（`pdm.md`）**: KOICA 様式そのまま — インパクト / アウトカム / アウトプット / 活動を横 4 列
  （要約、検証指標、検証手段、重要な前提）で整理した表。国際開発協力事業の基本成果物です。
- **セオリー・オブ・チェンジ図（`toc.md`）**: 活動からアウトプットへ、アウトプットからアウトカムへの
  つながりをノード図で示します。インパクトスタートアップ・社会貢献・非営利ではこれが基本成果物で、
  インパクトを測定するためにどのデータを自ら集めておくべきかも併せて教えてくれます。
- **モニタリング表（`monitoring.md`）**: 指標ごとに定義、算出式、ベースライン、目標値、根拠、データ源、
  測定時期、収集主体、分解基準を整理した表です。
- **事業予算書（`budget.md`、任意）**: 国際開発協力（PDM）では希望に応じて、活動別の細目・算出根拠
  （単価 × 数量 × 回数）・分担・一般管理費まで、PDM と連動した予算ドラフトも作成します。合計の計算と
  検証はスクリプトが決定論的に行います。

ベースラインと目標値は通常、現地調査を経て決まるため、最初は「後日確定（추후 확정）」として空けて
おき、値が決まった時点で埋めれば大丈夫です。

ファイルは次のように整理されます。**メイン成果物（`pdm.md` または `toc.md`）が `out/` 直下**にあり、
モニタリング表と元データは `out/details/` に置かれます。

```
out/
├── pdm.md          (または toc.md)   ← メイン成果物
└── details/
    ├── monitoring.md
    └── pdm.json    (各表の元になるソースデータ)
```

## 品質チェックの仕組み

仕上げの直前に、作成された PDM が KOICA ガイドラインの主要ルールを守っているか自己チェックします。
例えば：

- インパクトには個別の指標を入れない。
- アウトプットは 3〜4 個に整理する。
- アウトカムあたりの指標は 1〜2 個（最大 3 個）。
- すべての指標に測定方法（検証手段）が書かれている。
- 活動はアウトプットへ、アウトプットはアウトカムへ漏れなくつながっている。
- アウトカムは量的拡大ではなく対象の「行動変化」で表現する。

必須ルールは合格するまで一緒に磨き上げてから仕上げ、推奨ルール（SMART、CREAM、性別分解指標など）は
スコアで報告します。ルールの全リストとガイドラインのページ根拠は
[`skills/theory-of-change-agent/rules/koica-rules.md`](./skills/theory-of-change-agent/rules/koica-rules.md)
にまとめてあります。

## インストール

普段 **Claude デスクトップアプリや claude.ai** をお使いなら**方法 3**を、ターミナルツールの
**Claude Code** に慣れているなら**方法 1**が最も簡単です。

### 方法 1. Claude Code プラグイン（自動更新、推奨）

Claude Code 内で次の 2 行を入力するとインストールされます。

```
/plugin marketplace add IMPACT-SQUARE/theory-of-change-agent
/plugin install theory-of-change-agent@impact-square
```

後で最新版を取得するには：

```
/plugin marketplace update impact-square
/plugin update theory-of-change-agent
```

### 方法 2. 直接リンク（クイックインストール）

```bash
git clone git@github.com:IMPACT-SQUARE/theory-of-change-agent.git
ln -s "$(pwd)/theory-of-change-agent/skills/theory-of-change-agent" ~/.claude/skills/theory-of-change-agent
```

更新は `git pull` で。リンクを外すには `rm ~/.claude/skills/theory-of-change-agent` を実行します。
元のフォルダはそのまま残ります。

### 方法 3. zip ファイルのアップロード（Claude デスクトップ · Antigravity · claude.ai）

Claude Code ではなく**アプリで直接**使うには、スキルを zip に圧縮してアップロードします。

1. **zip の作成**: ダウンロードしたフォルダの `skills` フォルダ内で次を実行します。
   ```bash
   cd skills
   zip -r theory-of-change-agent.zip theory-of-change-agent \
     -x '*/.DS_Store' -x '*/out/*' -x '*/.omc/*' -x '*/__pycache__/*'
   ```
   （リポジトリに作成済みの `theory-of-change-agent.zip` があれば、それをそのまま使えます。）

2. **アップロード** — お使いのアプリに合わせて：
   - **Claude デスクトップ**: `Settings → Skills → Add → Upload` で `theory-of-change-agent.zip` を選択 → 有効化（Enable）。
   - **Antigravity**: Skills 画面で `theory-of-change-agent.zip` をアップロード。
   - **claude.ai（Web）**: `Settings → Capabilities → Skills → Upload`。

   > 前提: Claude アプリは**有料プラン + コード実行（code execution）機能オン**が必要です
   > （品質チェックが python3 で動きます）。メニュー名はバージョンによって多少異なります。

3. **使い方**: チャットに「セオリー・オブ・チェンジ エージェントで KOICA PDM を作って」のように
   書けば始まります。

#### セオリー・オブ・チェンジ図（Mermaid）が Antigravity で表示されないとき

`out/toc.md` の図は **Mermaid `flowchart`** で描かれます。Antigravity（および Mermaid プレビューの
ない VS Code）は初期状態で ```mermaid``` ブロックを**図としてレンダリングしない**ため、コードのように
見えることがあります。解決策は 2 つ：

- **（推奨）Mermaid プレビュー拡張のインストール** — Antigravity は VS Code 系なので、拡張マーケット
  （**Open VSX**）から **`Markdown Preview Mermaid Support`**（パブリッシャー `bierner`、
  id `bierner.markdown-mermaid`）をインストールすると、`toc.md` のプレビューで図が表示されます。
  *（拡張はアプリ側で手動インストールが必要です — スキルが自動インストールすることはできません。）*
- **インストールせずそのまま読む** — スキルは Mermaid ブロックの**すぐ下に同じ内容のテキスト因果フロー
  （→ 矢印）**を常に併記します。拡張がなくても事業のロジックはそのテキストで読めます。
   完成した成果物はチャットのダウンロードリンクで受け取ります。

> アプリにアップロードしたスキルは**自動更新されません**。内容が変わったら新しい zip を
> 再アップロードしてください。自動更新が必要なら**方法 1（Claude Code プラグイン）**をお使いください。

詳しい案内は
[`skills/theory-of-change-agent/INSTALL-desktop.md`](./skills/theory-of-change-agent/INSTALL-desktop.md)
にあります。

### 必要なもの

- **Claude**（Claude Code、デスクトップアプリ、Web のいずれでも）
- **`python3`**: 品質チェックの実行に使います。デスクトップ/Web のコード実行環境には標準で含まれています。

## データポリシー

- このリポジトリの `docs/` にある PDF は公開文書（KOICA PDM ガイドライン、Theory of Change 資料）のみです。
- 品質チェック用のサンプルデータ（`benchmark/`）は、実名と実際の金額をすべて除いた架空の PDM で、
  構造だけを残しています。
- 実際の事業 PDM の原本はこのリポジトリには含めず、別途非公開で保管しています。
