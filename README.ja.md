# Theory of Change Agent

[English](README.md) · [한국어](README.ko.md) · **日本語** · [Tiếng Việt](README.vi.md)

成果マトリックスの作成は、事業を構想すること自体より難しい場合が少なくありません。課題も活動も
目指す成果も分かっているのに、それを筋の通った PDM に落とし込むのは、また別の作業だからです。

**Theory of Change Agent は、この作業を対話形式で進めるツールです。** 質問を一つずつ投げかけ、
回答を結果連鎖（社会問題 → 活動 → アウトプット → アウトカム → インパクト）にまとめていきます。
国際開発協力事業であれば **KOICA ガイドライン準拠の PDM（Project Design Matrix）** を、インパクト
スタートアップ・CSR・非営利であれば**変化の理論（Theory of Change）図**を作成します。Claude などの
AI アシスタント上で動作し、話しかけた言語で応答します。

テンプレートを埋めるだけのツールではありません。アウトプットがアウトカムとして書かれているような
論理の弱点を指摘します。しかもチェックの根拠はたどれます。決定論的な品質ゲートは検出ベンチマークで
**18/18** を記録し、各ルールは **KOICA ガイドライン（全 36 ページ）の該当ページ**を引用し、予算
モジュールは KOICA 型の実予算書 2 種の合計を**ウォン単位まで再現**しました。

**こんな方に向いています。**

- KOICA などの公募事業に応募し、**PDM を初めて書く方**
- 下書きを**審査の前にガイドラインと照合しておきたい方**
- 初期のインパクトモデルを整理したい**起業家、CSR・非営利チーム**

## クイックスタート

1. スキルをインストールします。30 秒ほどで終わります。
2. *「Theory of Change Agent で PDM を作って」* と話しかけます。事業計画書や年次報告書を添付しても構いません。
3. 質問に答え、内容が固まったら**「確定」**と伝えます。
4. `out/` で結果を確認します。国際開発協力は `pdm.md`、それ以外は `toc.md` です。

## 進め方を選ぶ

インタビューは問題から出発して原因をたどり、目指す変化を測定可能な形に整えます。
最終的な成果物は用途によって変わります。

| 状況 | 作成される成果物 | 用意するとよいもの |
|---|---|---|
| 国際開発協力プロジェクト | KOICA ガイドライン準拠 PDM | アイデア、提案書、既存 PDM の下書き |
| インパクトスタートアップ・新規事業 | 変化の理論図 | 事業計画書、または解決したい問題 |
| CSR・ESG プロジェクト | 変化の理論図 | 事業概要書、または提案書 |
| 非営利プログラム・年次報告 | 変化の理論図 | 年次報告書。組織全体か一つのプロジェクトかを選択 |

新しい PDM は、確定の前に品質ゲートを通過する必要があります。承認済みの PDM は監査モードで点検
でき、文書には手を加えず、ガイドラインとの相違点だけを報告します。

## 実際の動き

```
あなた:  Theory of Change Agent で PDM を作って。
Agent:  どんなお仕事ですか？
        ① 国際開発協力（PDM） ② インパクトスタートアップ
        ③ CSR / ESG          ④ 非営利
あなた:  ①。ラオスの母子保健事業です。
Agent:  [質問 1 · 社会問題] この事業が解決しようとする問題は何ですか？
        誰が困っていて、原因は何でしょうか。
あなた:  農村部の妊産婦が自宅で出産しています。保健施設が遠く、
        信頼もされていません。
Agent:  社会問題として扱えます。構造的で、多くの人に影響し、実際の
        被害があります。表れている現象は高い自宅分娩率、原因は施設への
        アクセスと信頼の不足です。とすればアウトカムは「より多くの
        妊産婦が施設で出産する」のような状態の変化として設定できます。
        「研修回数の拡大」ではなく。意図している変化と合っていますか？
        …
        [活動、アウトプット、指標、前提を一つずつ質問し、
         回答のたびに結果連鎖へつなぎます。]
        …
あなた:  確定
Agent:  ✔ 品質ゲート通過（重要ルール 8/8）。
        ⚠ アウトカム 2 はアウトプットの拡大に近い書き方です。「教育の
          拡大」は回数を数えるもので、行動の変化ではありません。
          修正案: …
        ◇ 指標 1-1 に近い IRIS+ 指標を提案します。参考情報であり、
          公式なマッピングではありません。
        結果: out/pdm.md
```

インタビューは、活動・アウトプットと、事業が目指す変化とを区別します。審査でよく指摘される問題を、
提出前に洗い出すのに役立ちます。

## 仕組み

**インタビュー → 結果連鎖の構築 → 検証 → ゲート → レンダリング。** 各ステップは前のステップの
成果を引き継ぎます。

インタビューは変化の理論の考え方に沿って進みます。まず、構造的で、多くの人に影響し、実際の被害を
生む**社会問題**を定義します。次に、表れている現象と原因を結びつけ、その原因を解消する変化を
アウトカムとして定めます。アウトプットや漠然とした便益の記述は、アウトカムとしては扱いません。

年次報告書や事業計画書のような組織単位の文書を入力すると、その文書が一つのプロジェクトを扱って
いるのか複数なのかを先に確認します。**組織全体**をミッションとプロジェクトのつながりとして俯瞰
することも、**一つのプロジェクト**に絞ることもできます。

| 成果物 | 内容 |
|---|---|
| `pdm.md` | KOICA 4×4 PDM マトリックス: インパクト/アウトカム/アウトプット/活動 × 要約/指標/検証手段/重要な前提 |
| `toc.md` | 変化の理論図。Mermaid 非対応ビューア向けのテキスト版と、インパクト測定のために集めておくべきデータの案内付き |
| `details/monitoring.md` | 指標ごとの測定計画: 定義、算出式、ベースライン、目標値、出所、時期、収集主体、分解基準 |
| `budget.md` *(任意)* | PDM 連動の予算書: 活動別の細目、算出根拠（単価×数量×回数）、資金分担、年次別合計 |
| `details/pdm.json` | 上のすべてのビューをレンダリングする元データ |

会話だけで始めることも、PDF や**韓国語 HWP ファイル（.hwp/.hwpx）**を入力することもできます。
同梱の抽出器は外部依存がないため、アプリのサンドボックス内でも動作します。

## 自分で確かめられる検証

チェックはコードとして実装されており、このリポジトリで確認できます。

- **決定論的品質ゲート:** インパクトに指標を置かない、アウトプットは 3〜4 個、指標数の上限、
  検証手段の必須化、孤児ノードの禁止、アウトプットは名詞形といった KOICA の重要ルール 8 個を
  純 Python で検査します。[違反埋め込みベンチマーク](./skills/theory-of-change-agent/benchmark/)で
  **検出精度 18/18** を記録しました。
- **ページ引用ルール:** [`koica-rules.md`](./skills/theory-of-change-agent/rules/koica-rules.md) の
  すべてのルールが、KOICA 公式ガイドラインの該当ページを引用しています。
- **アウトカムの検討:** 意図した変化が原因を実際に解消するかを論理的に確認します。アウトカム指標は
  **IRIS+ のインパクト指標 593 個**の中から最も近いものと照合します。参考情報であり、公式な
  マッピングではありません。
- **予算計算はスクリプトで:** すべての合計、比率、資金分担、一般管理費の上限をスクリプトが計算・
  検証します。KOICA 型の実予算書 2 種を使った 3 回の検証で合計をウォン単位まで再現し、その過程で
  ルールの不具合 3 件を発見して修正しました。
- **推奨ルール:** SMART、CREAM、性別分解指標はスコアとして報告するだけです。反映するかどうかは
  利用者が決めます。

## インストール: 30 秒

**必要なもの:** Claude Code、Claude デスクトップ、claude.ai のいずれかと `python3`。デスクトップと
Web のコード実行サンドボックスには Python が含まれています。

### Claude Code（自動更新、推奨）

```
/plugin marketplace add IMPACT-SQUARE/theory-of-change-agent
/plugin install theory-of-change-agent@impact-square
```

更新は `/plugin update theory-of-change-agent`。

### Claude デスクトップ · Antigravity · claude.ai（zip アップロード）

1. スキルを zip 化します。ビルド済みの `theory-of-change-agent.zip` があればそのまま使えます。
   ```bash
   cd skills && zip -r theory-of-change-agent.zip theory-of-change-agent \
     -x '*/.DS_Store' -x '*/out/*' -x '*/.omc/*' -x '*/__pycache__/*'
   ```
2. アップロードします。**Claude デスクトップ**は `Settings → Skills → Add → Upload`、**Antigravity**
   は Skills 画面、**claude.ai** は `Settings → Capabilities → Skills → Upload`。有料プランとコード
   実行機能が必要です。
3. チャットに *「Theory of Change Agent で KOICA PDM を作って」* と書けば始まります。

> アプリにアップロードしたスキルは自動では更新されません。内容が変わったら新しい zip を再アップ
> ロードしてください。Antigravity で図がコードのまま表示される場合は、Open VSX から
> `bierner.markdown-mermaid` をインストールするか、併記されるテキスト版をお読みください。詳細:
> [`INSTALL-desktop.md`](./skills/theory-of-change-agent/INSTALL-desktop.md)。

## データポリシー

- `docs/` には公開文書のみを置いています（KOICA PDM ガイドライン、Theory of Change 資料）。
- `benchmark/` の PDM は架空のデータです。構造は保ちつつ、実名と実際の金額は取り除いています。
- 実際の事業の PDM・予算書の原本は、このリポジトリには保存しません。
